# Contributing

Contributions are welcome when they make the method safer, clearer, more
portable, or easier to reuse.

Good contributions:

- improve validation gates
- add sanitized examples
- add OS-agnostic templates
- document maintainer-feedback lessons
- improve privacy defaults
- add optional tools with clear data contracts

Avoid:

- vendor lock-in
- mass-comment automation
- raw transcript publishing
- private queue details
- scraping or probing maintainers without consent

Before opening a PR, check that the change still fits the core philosophy:
help maintainers first.
