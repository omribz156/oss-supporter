# Publication Boundary

The public repo is the method. The private workbench is the live operation.

## Public

- playbooks
- templates
- sanitized case studies
- aggregate metrics
- public PR/comment links
- reusable review lessons
- tools with privacy-safe defaults

## Private

- raw chats and prompts
- raw session logs
- local clone folders
- credentials
- account or CLA details beyond public bot status
- private queues
- target strategy before public action
- personal identity details

## Sanitization Rules

- Keep public links when the action is already public.
- Remove private notes and local file paths.
- Remove raw model output.
- Summarize instead of copying transcripts.
- Publish aggregate token and impact numbers, not raw logs.
- Review generated exports before committing.
