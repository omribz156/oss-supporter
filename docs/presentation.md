# Presentation

The public repo should feel like a real open project, not a private workbench
dump.

## Shape

- Banner first: name, philosophy, and loop.
- Badges second: license, portability, maintainer-first.
- Start Here list: the minimum path for a new operator.
- Operating model: one page, no tool lock-in.
- Templates: copyable and boring.
- Impact: aggregate, public, privacy-safe.

## Visual Rules

- Use visuals to explain the system, not decorate noise.
- Keep screenshots and diagrams generic.
- Do not expose private paths, session IDs, target queues, or live notes.
- Prefer SVG diagrams for portability.
- Use public PR/comment links only when already public.

## Tone

Clear, direct, slightly alive.

Good:

```text
Use spare tokens for good.
```

Avoid:

```text
Autonomous AI swarm revolutionizes OSS forever.
```
